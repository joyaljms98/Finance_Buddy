📂 1. The Folder Structure
FastAPI doesn't force you into a specific folder structure (unlike Next.js or Django), so adopting a clean, modular structure is crucial for keeping your code organized as Finance Buddy grows.

app/api/ (Endpoint Routers):
This folder holds the actual "URLs" (endpoints or routes) that your frontend will call.
Instead of putting 100 API endpoints in one massive file, you break them up by feature.
Example: All login/signup code goes in api/auth.py. All cashbook transaction logic goes in api/cashbook.py. FastAPI has a feature called APIRouter that lets you easily stitch these separate files together.

app/models/ (Pydantic & DB Schemas):
This is where you define the "shape" of your data.
For example, you define a User model here that strictly says: "A user must have a string Name, a string Email, and an integer Age." If the frontend accidentally sends text for the age, the model automatically catches the error before it ever reaches the database.

app/core/ (Configurations & Security):
This folder is the "engine room" of the backend.
It contains security.py (which houses the math that scrambles passwords) and config.py (which securely loads sensitive variables like your MongoDB password from hidden .env files).

app/db/ (Database Connection):
This holds the code that actually reaches out across the internet and connects to your MongoDB Atlas cluster. When your server starts up, this folder ensures the database connection is alive and ready to accept queries.



🛡️ 2. Authentication Strategy (JWTs)
Right now, your frontend checks a fake localStorage to see if someone is logged in. In a real app, users can easily fake that. We are replacing it with JWT (JSON Web Tokens).

What is a JWT? When a user successfully logs in with the right email and password, your backend creates a unique, cryptographically signed "Token" (a long string of characters). The frontend saves this token and attaches it to every future request it makes to the backend (like a VIP pass).
python-jose: This is the Python library we use to actually generate those secure JWTs and mathematically verify them when the frontend sends them back.

passlib & bcrypt: When a user creates an account, we never save their password (like "password123") in plain text in the database. passlib uses the bcrypt algorithm to scramble that password into an unreadable string (a hash). When they log in, it scrambles their input again and compares the two scrambles. Even if someone hacks your database, they won't know the users' real passwords!

Depends() (Active Protection): FastAPI has a brilliant feature called Dependency Injection. When you build an endpoint like /api/cashbook/my-transactions, you add Depends(get_current_user). Before running any code in that endpoint, FastAPI will automatically grab the VIP Token from the request, use python-jose to verify it's legit, and inject the specific user's info into the function. If the token is fake or expired, it automatically throws a 401 Unauthorized error block.



🗄️ 3. Database Modeling Strategy (Motor)
MongoDB has an official Python library called PyMongo. However, we are using Motor.

Why Motor? Remember how FastAPI is Asynchronous (super fast because it doesn't wait for one task to finish before starting another)? Standard PyMongo is synchronous (slow, blocking). Motor is a special async version of PyMongo built specifically to talk to MongoDB without slowing down FastAPI's lightning engine.



📦 4. The Python Dependencies (The "npm install" of Python)
When we start the backend, we run pip install on these libraries. Here is exactly what each one does for Finance Buddy:

fastapi: The core web framework itself. It handles the web server, the routing, and auto-generates your API documentation (Swagger UI).
uvicorn: FastAPI is just the framework; it doesn't know how to actually run on a computer network. uvicorn is the high-performance "ASGI server" that actually runs your FastAPI code and listens for internet traffic on port 8000.
pydantic: The data-validation superhero. It guarantees that the JSON data your frontend sends mathematically matches the exact shapes you define in your models/ folder.
motor: As explained, the asynchronous bridge that lets FastAPI talk to your MongoDB Atlas cluster at maximum speed.
python-jose: Builds and verifies the VIP JWT login tokens.
passlib (with bcrypt): Safely scrambles and hashes user passwords.
python-dotenv: A utility that allows Python to read hidden .env files. This is how we keep your secret MongoDB URI and JWT Secret Keys out of the public Github code!
chromadb: The local, entirely free Vector Database. We will use this specifically to store the high-dimensional embeddings generated by LLMs during the future RAG chatbot phase, keeping our MongoDB M0 free-tier purely for traditional user and financial data!
